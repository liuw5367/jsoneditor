# JSON Editor

一个在浏览器中运行的 JSON 编辑与对比工具。编辑内容保存在本地，支持格式化、压缩、文件目录管理和两份 JSON 的差异查看。

## 功能

- 使用文本、树形或表格模式编辑 JSON
- 美化 JSON、压缩为单行、展开或收起树形内容
- 自动将当前草稿保存到浏览器本地存储
- 选择本地目录并新建、打开、重命名、保存或删除 JSON 文件
- 对比两份 JSON，统计并筛选新增、删除和修改项
- 交换对比内容，或将右侧内容应用到编辑器

## 环境要求

- Node.js 18 或更高版本
- pnpm 9.15.9
- Chromium 系浏览器，用于目录选择和文件写入

目录操作依赖 [File System Access API](https://developer.mozilla.org/docs/Web/API/File_System_API)。不支持该 API 的浏览器仍可编辑 JSON，并通过下载方式保存内容，但无法直接管理本地目录。

## 本地运行

安装依赖：

```bash
pnpm install
```

启动开发服务器：

```bash
pnpm dev
```

默认访问地址为 `http://127.0.0.1:5173/`。

构建并预览生产版本：

```bash
pnpm build
pnpm preview
```

## 使用说明

### 编辑 JSON

编辑器顶部可切换文本、树形和表格模式。常用操作位于模式切换之后：

1. 美化 JSON
2. 压缩为一行
3. 全部展开
4. 全部收起

编辑内容变化后会自动写入浏览器的 `localStorage`。草稿只用于恢复当前编辑内容，不会自动写入本地文件。

### 管理本地文件

点击顶栏中的目录区域选择一个本地目录。授权后，应用只列出其中的 `.json` 文件，并提供打开、重命名和删除操作。

- 未选择目录时，“新建保存”会下载 JSON 文件。
- 已选择目录但未打开文件时，“新建保存”会在该目录创建文件。
- 打开已有文件后，可以直接保存或删除当前文件。

浏览器会要求用户明确授权目录读写权限。权限被撤销后，需要重新授权才能继续访问。

### 对比 JSON

切换到“对比”模式后，可以从已授权目录载入左右两份文件，也可以直接编辑两侧内容。页面会统计新增、删除和修改数量，点击统计项可以查看对应的 JSON 路径。

“交换两侧”会交换原始与目标内容；“应用右侧到编辑器”会将右侧内容复制到编辑模式，但不会自动保存到文件。

## 快捷键

| 操作 | 快捷键 |
| --- | --- |
| 美化 JSON | `Ctrl + I` |
| 压缩为一行 | `Ctrl + Shift + I` |
| 搜索 | `Ctrl + F` |
| 撤销 | `Ctrl + Z` |
| 重做 | `Ctrl + Shift + Z` |

快捷键由 `vanilla-jsoneditor` 提供，实际按键提示以编辑器工具栏为准。

## 数据与隐私

应用不包含服务端接口。JSON 解析、编辑和对比都在当前浏览器中完成：

- 当前草稿保存在浏览器 `localStorage` 中。
- 已选择的目录句柄保存在浏览器 IndexedDB 中，后续访问仍受浏览器权限控制。
- 文件内容只在用户执行打开、保存或删除操作时访问。
- 应用不会主动上传 JSON 内容。
