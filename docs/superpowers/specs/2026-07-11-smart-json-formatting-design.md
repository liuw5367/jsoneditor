# 智能 JSON 格式化设计

## 目标

为文本模式增加与 JSONEditor Online 行为接近的“智能格式化”操作。它在保持 JSON 可读性的同时，将短数组和简单对象保留在一行。用户可以在独立设置弹窗中调整最大行宽。

## 交互

- 编辑器工具栏顺序为：美化 JSON、智能格式化、压缩、分隔线、全部展开、全部收起。
- 智能格式化快捷键为 `Ctrl/Cmd + J`。
- 应用顶栏增加“设置”图标按钮，打开独立设置弹窗。
- 设置项“智能格式化最大行宽”允许输入 40–500，默认值为 120。
- 保存设置后写入 `localStorage`；取消不会修改当前设置。
- 输入非法时在弹窗内显示错误，并阻止保存。
- JSON 无效或格式化失败时不修改编辑内容，通过现有状态区域提示失败原因。

## 实现

- 使用 MIT 许可的 `fracturedjsonjs`，不自行实现格式化算法。
- 智能格式化逻辑封装为可测试的异步函数，输入 JSON 文本、缩进和最大行宽，输出格式化文本。
- 设置的解析、范围校验和持久化封装为纯函数；存储不可用时回退到默认值。
- 通过 `vanilla-jsoneditor` 的 `onRenderMenu` 插入自定义按钮，并复用编辑器实例更新内容。
- 不升级 `vanilla-jsoneditor`，不更改文件保存和对比模式的数据流程。

## 验证

- 单元测试覆盖默认设置、边界值、非法值、存储失败、智能格式化输出和菜单顺序。
- 手工验证按钮、快捷键、弹窗保存与取消、无效 JSON 提示及窄屏布局。
- 运行 `pnpm test`、`pnpm build` 和 `git diff --check`。
